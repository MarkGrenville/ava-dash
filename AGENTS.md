# Ava Dashboard — Agent Notes

## What is Ava Dashboard?
A task management dashboard for the Ava AI bot. Ava is a Claude Code-powered bot running on a Mac Mini that handles tasks via email — research, web scraping, automation. This dashboard lets the human operator monitor what Ava is working on, and gives Ava a structured way to plan multi-step work.

The key problem this solves: Ava was a black box. Tasks went in, results sometimes came back. This system gives Ava a way to break work into projects with ordered tasks, track progress through them, and report back when everything is done.

## Tech Stack
- **Frontend:** SvelteKit 2 + Svelte 5 (static adapter → `build/`)
- **Styling:** Tailwind CSS v4 via `@tailwindcss/vite` plugin, dark theme
- **Backend:** Firebase Cloud Functions v2 (Express API with Swagger)
- **Database:** Firestore (emulators for local dev)
- **Auth:** Firebase Auth (Email + Google) for dashboard, API keys for Claude Code
- **Package Manager:** pnpm
- **Process Manager:** PM2 (ecosystem.config.cjs)

## Two Auth Paths
1. **Dashboard (human):** Firebase Auth (Email/Google) → Firestore security rules
2. **API (Claude Code):** API key in `Authorization: Bearer <key>` header → Cloud Function middleware validates against `apiKeys` collection

## Data Model (summary)
- `projects/{id}` — Title, description, status, task counts, metadata blob
- `projects/{id}/tasks/{taskId}` — Ordered subtasks with status, result, dependencies, `scheduledFor` for time-specific tasks
- `cronJobs/{id}` — Scheduled jobs with cron expressions
- `activityLog/{id}` — Immutable audit log with `source`, `tokensUsed`, `tokensRemaining`, `projectId`, `taskId` fields
- `apiKeys/{id}` — Hashed API keys for Claude Code auth
- `users/{uid}` — User profile with admin role

### Scheduled Tasks
Tasks can have a `scheduledFor` field (ISO datetime). When set, status is `scheduled`. A cron job calls `POST /tasks/check-schedule` every minute to activate due tasks (→ `pending`). Endpoints: `GET /tasks/due`, `GET /tasks/upcoming`, `POST /tasks/check-schedule`.

### Audit Log
`POST /activity` lets Ava log actions with source tracking (`cron`, `user_request`, `email`, `api`, `system`, `scheduled`) and token usage (`tokensUsed`, `tokensRemaining`). The dashboard shows a summary with token burn rate.

See `docs/models.md` for full field-level schemas and `docs/AVA_MDP.md` for Ava's behavioral guidelines.

## Operational Notes
- The project uses PM2. Check `pm2 list` and `pm2 logs ava-dash-frontend --lines 100 --nostream`.
- Never deploy — the user deploys manually via `./scripts/deploy.sh`.
- Never run git commits — the user handles version control.
- Never kill or restart the emulator — the user will do it if needed.
- Emulator backup runs every 2 minutes via `ava-dash-backup` PM2 process.
- Business logic lives in Cloud Functions, not SvelteKit server routes.
